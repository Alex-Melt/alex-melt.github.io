# 使用介绍

个人博客框架 hexo, 托管于 GitHub，博客网站使用 Github 自带的 Page 或者 netlify 生成，国内访问采用 cloudflare 进行 CDN 加速

# hexo 框架

介绍：基于 nodejs 的静态博客网站生成器，支持 markdown 语法，能够快速生成静态 html

npm 配置

  • 简介：npm 是 JavaScript 运行时环境 Node.js 的默认包管理器，用于管理依赖包的下载与卸载
  • 检查 npm 的路径
#使用命令查看位置(默认在c盘)
npm config get prefix
npm config get cache

#建议修改到其他盘
在Node.js文件下创建node_cache文件夹
使用命令：
npm config set prefix "F:\environment\NodeJs"
npm config set cache "F:\environment\NodeJs\node_cache"

# 注意 (万分重要！！！)

在对后序所有地方修改时都需要注意：空格

安装 hexo

npm install hexo -g #-g表示全局安装, npm默认为当前项目安装

hexo 基本命令

npm update hexo -g  #升级 
hexo init <folder>  #执行init命令初始化hexo到你指定的目录
hexo n "我的博客" == hexo new "我的博客" #新建文章
hexo generate       #自动根据当前目录下文件,生成静态网页 hexo g
hexo server         #运行本地服务 hexo s
hexo clean          #清理
hexo deploy         #部署 hexo d

hexo server         #Hexo会监视文件变动并自动更新，无须重启服务器
hexo server -s      #静态模式
hexo server -p 5000 #更改端口
hexo server -i 192.168.1.1 #自定义 IP

# 初始化

方式一：

  • 选择合适的位置创建博客所在位置 ，假设创建的文件夹为：blog
  • 进入该文件夹

方式二：

执行命令后生成的文件

  • _config.yml
    • 全局配置文件，例如网站的名称，副标题，描述，作者，语言，主题等
  • scaffolds
    • 骨架文件，生成新页面或者新博客的模板，当 hexo 生成新博客时，会在这里初始化
  • source
    • 存放 markdown 源文件，，里面的 _post 文件夹也就是笔记存放的位置
  • themes
    • 网站主题目录，可以在 https://hexo.io/themes/ 下找到相关主题

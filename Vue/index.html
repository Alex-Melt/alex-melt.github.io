# Vue 基础

是一款用于构建用户界面的 JavaScript 框架，基于标准 HTML，CSS，JavaScript 构建，提供了一套声明式的，组件化的编程模型

Vue 的特点

  • 采用组件化模式，一个 vue 文件就是一个组件，提高代码复用率，代码更好维护
  • 声明式编码，无需直接操作 DOM，将 Json 文件放入前端容器中，生成效果
  • 使用优秀的虚拟 DOM 和 diff 算法

安装

  • 使用 cdn 加速
  • 引入本地的 vue.js

# 核心功能

  • 声明式渲染：Vue 基于标准 HTML 拓展了一套模板语法，可以声明式的描述 HTML 与 JavaScript 之间的关系
  • 响应性：Vue 会自动跟踪 JavaScript 状态变化并在改变发生时响应式的更新 DOM

# 渐进式框架

  • 无需构建步骤，渐进式增强静态的 HTML

  • 在任何页面中作为 Web Components 嵌入

  • 单页应用 (SPA)

  • 全栈 / 服务端渲染 (SSR)

  • Jamstack / 静态站点生成 (SSG)

  • 开发桌面端、移动端、WebGL，甚至是命令行终端中的界面

1. 创建一个vue实例，传入一个配置对象

7. 一旦data中的数据发生改变，模板中用到该数据的地方也会自动更新

# 事件处理

事件的基本使用：
    1. 使用v-on:xxx或者@xxx 绑定事件
    2. 事件的回调需要配置在methods中，都会显示在vm中，不放在data中是因为在data中会数据劫持，即会自动给data中的属性加上代理，而函数不需要，增加了不必要的负担
    3. methods中配置的函数，不需要使用箭头函数，因为箭头函数的this指的是window对象，而不是vue实例对象或者组件实例对象
    4. methods中配置的函数，都是被vue所i管理的对象，this指向的是vue实例
    5. @click="demo"和@click="demo($event)"两者效果一致，但是后者可以传递参数
vue中的事件修饰符
    1. prevent：阻止默认事件
    2. stop：阻止事件冒泡
    3. once：事件只触发一次
    4. capture：使用事件的捕获模式
    5. self：只有event.target是当前操作的元素时才触发事件
    6. passive：事件的默认行为是立即执行，无需等待事件回调执行完毕
键盘事件：
    1. vue中常用的按键别名：
        0. 换行 --》tab（特殊，必须配合keydown使用）
        1. 回车 --》enter
        2. 删除 --》delete(捕获删除和退格两个按键)
        3. 退出 --》esc
        4. 空格 --》space
        5. 上 --》up
        6. 下 --》down
        7. 左 --》left
        8. 右 --》right
    2. vue未提供别名的摁键，可以使用按键原始的key值去绑定，但需要转化为kebab-case
    3. 系统修饰键（特殊）：ctrl，alt，shift，meta
        1. 配合keydown正常触发事件
        2. 配合keyup使用，按下修饰键的同时，再按下其他键，随后释放其他键，事件才会被触发
    4. 也可以使用keyCode查看

# 列表渲染

列表渲染:
v-for指令：
    1. 用于展示列表数据
    2. 语法：v-for="(item,index) in xxx" :key="yyy"
    3. 可遍历：数组，对象，字符串，指定次数
react、vue中的key的作用，原理是什么：
    1. 虚拟DOM中key的作用：
        key是虚拟DOM对象的标识，当状态中的数据发生改变时，vue根据新数据生成虚拟DOM，
        随后vue进行新虚拟DOM与旧虚拟DOM进行差异比较
    2. 对比规则：
        1. 旧虚拟DOM找到了与新虚拟DOM相同的Key
            若虚拟DOM中的内容没变，直接复用原来的真实DOM
            若内容改变了，则直接生成新的DOM。随后替换掉页面中真实的DOM
        2. 旧虚拟DOM没有找到与新虚拟DOM相同的key：
            创建新的真实DOM，渲染到页面
    3. 用index作为key会引发的问题：
        1. 若对数据进行：逆序添加，逆序删除等破坏顺序操作，产生没必要的真实DOM更新，效率低
        2. 如果结构中还包括输入类的DOM，例如input，会产生错误的DOM，顺序会错乱，因为进行虚拟DOM算法比较时，input都显示为没有数据，默认相等
    4. 开发中选择key：
        1. 最好使用每条数据的唯一标识id，shoujihao，身份证，作为key
        2. 如果不存在对数据的逆序添加，逆序删除等破坏顺序操作，仅用于渲染，可以使用index作为key

# 数据劫持的原理

vue数据劫持的原理：
    1. vue会监视data中所有层次的数据
    2. 监测对象中的数据：
        通过setter实现监测，且要在new vue时就传入要监测的数据
        对象中后追加的属性，vue默认不做响应式处理，即通过简单的赋值无效
        需要做响应式处理，使用：
            Vue.set(target,propertyName/index,value)或者
            vm.$set(target,propertyName/index,value)
    3. 监测数组中的数据：
        通过包裹数组更新元素的方法实现：调用原生对应的方法进行数组的更新，重新解析模板，进行更新界面
        通过赋值改变的数组元素，不会生效，需要使用pop,push,splice,shift,unshift,sort,reverse,或者使用Vue.set(),vm.$set()

# 内置指令

接触的指令：
    v-bind：单向绑定数据，可以简写为 v-bind:href="" -> :href=""
    v-model：双向数据绑定，可以简写为 v-model:value="" -> v-model=""
    v-for：遍历数组/对象，字符串
     v-on：绑定事件监听，可以简写为 v-on:xxx -> @xxx
    v-if：条件渲染（动态控制节点是否存在）
    v-else：条件渲染（动态控制节点是否存在）
    v-show：条件渲染（动态控制节点是否展示）
v-text: 向其所在的结点中渲染文本内容，v-text会替换掉标签中的所有内容，而且只会输出字符串，不会解析
v-html：向其所在的结点中渲染文本内容，v-text会替换掉标签中的所有内容，而且可以解析字符串的标签
    问题：在网站动态渲染html存在安全性容易导致XSS攻击，例如点击链接时，可能将本网站的cookie携带过去，造成信息泄露
v-cloak：(没有值)
    1. 本质是一个特殊属性，vue创建实例完毕并接管容器后，会删除v-cloak属性
    2. 使用css配合v-cloak可以解决网速慢时页面展示出{{xxx}}的问题
v-once：(没有值)
    1. v-once所在的结点在初次动态渲染后就视为静态内容了
    2. 以后数据的改变不会引起v-once所在结构的更新，可以用于优化性能
v-pre：跳过其所在节点的编译过程，没有使用插值语法的结点，会加快编译

# 自定义组件

1. 子组件向父组件传递：
    1. 父组件发送函数，子组件通过props接收后，返回信息给父组件
    2. 通过父组件自定义绑定事件，子组件触发事件，返回数据：
        1.使用v-on：
            父组件：<school v-on:atguigu="demo"/>，给组件的实例对象vc身上绑定了事件，并定义demo函数
            子组件：点击绑定事件，定义函数使用this.$emit($event,this.message)触发事件，返回给父组件数据，第一个参数代表触发的哪个事件，即atguigu，第二个参数代表返回的数据
        2.使用ref属性：
            父组件：<school ref="student"/>,写钩子mounted(){this.$refs.student.$on('atguigu',this.function)}
            子组件：点击绑定事件，定义函数使用this.$emit($event,this.message)触发事件，返回给父组件数据，第一个参数代表触发的哪个事件，即atguigu，第二个参数代表返回的数据
2. 解除绑定事件：
    解除一个绑定事件：this.$off($event)
    解除多个绑定事件：this.$off([$event,$event,$event])
    解除所有绑定事件：this.$off()
3. 绑定事件：
    绑定事件：this.$emit($event,this.message)
4. 销毁事件：实例对象销毁后，自定义事件不再生效
5. 对自定义组件实现原生态绑定事件 <student @click.native="demo"/>
注意：
    1.this.$refs.student.$on 中的第二个参数可以使用虚拟函数，
        1. ()=>{}：其中的this指的时当前vue对象
        2. function(){},此时特殊：触发事件的回调函数中的this指的是student
        3. 调用methods的数据

# 全局事件总线

1. 实行任意组件间的通信
2. 安装全局事件总线：
    new Vue({
        ...
        beforeCreate(){
            Vue.prototype.$bus=this
        },
        ...
    })
3. 使用全局事件总线
    1. 接受数据：A组件想接受数据，则在A组件中给$bus绑定自定义事件，事件的回调留在A组件自身，三种方式
        1. ()=>{}：其中的this指的时当前vue对象
        2. function(){},此时特殊：触发事件的回调函数中的this指的是student
        3. 调用methods的数据
        methods(){
            demo(data){...}
        },
        mounted(){
            this.$bus.$on('xxx',this.demo)
        }
    2. 提供数据：this.$bus.$emit('xxx',数据)
4. 在组件的beforeDestroy钩子中用$off解除绑定事件
